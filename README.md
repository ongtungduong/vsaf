- **Planning** вҖ” AI agents (Analyst, PM, Architect) clarify requirements; OpenSpec turns them into verifiable specs and task lists.
- **Code Intelligence** вҖ” GitNexus answers "what breaks if I change X?" via MCP; Graphify provides visual dependency maps and path tracing.
- **Memory** вҖ” claude-mem auto-captures sessions (zero config); MemPalace stores deliberate knowledge like architecture decisions (verbatim, lossless).
- **Implementation** вҖ” Claude Code writes code; Superpowers provides brainstorm/plan/TDD/review workflow; ECC cherry-pick runs security scans and hooks in the background.

## Prerequisites

- Node.js >= 18, npm
- Python >= 3.10, pip3, pipx
- Git, jq
- Claude Code with active subscription ($20/mo)
- Ubuntu 24.x or macOS

## Quickstart

```bash
git clone <this-repo> && cd vsaf
make setup          # Install all 8 tools (idempotent вҖ” safe to re-run)
make index          # Build knowledge graph
make status         # Verify everything is working
```

One manual post-setup step (requires an interactive Claude Code session):

```
/plugin install superpowers@claude-plugins-official
```

Restart Claude Code after installing Superpowers.

## Daily Operations

Run `make help` for the full list. Key commands:

| Command | What It Does |
|---|---|
| `make setup` | Install or update all tools |
| `make index` | Re-index codebase (GitNexus + Graphify) вҖ” run after every merge |
| `make scan` | AgentShield security scan (102 rules) |
| `make scan-deep` | AgentShield deep scan (Opus model + streaming) |
| `make verify` | Check implementation against OpenSpec specs (review Layer 2) |
| `make review` | Full 3-layer review coordinator |
| `make archive` | Archive completed specs + re-index (run after merge) |
| `make status` | Show status of all installed tools |
| `make mine` | Extract decisions from conversations into MemPalace (run weekly) |
| `make clean` | Remove GitNexus index (asks for confirmation) |

## 10-Step Workflow

Every feature follows this cycle. Bug fixes ("Quick Flow") skip steps 3вҖ“5.

| Step | What Happens | Key Command |
|------|-------------|-------------|
| 0 | One-time setup | `make setup` |
| 1 | Understand the codebase | Read `graphify-out/GRAPH_REPORT.md`, query MemPalace |
| 2 | Scope the work | `*agent analyst`, `*workflow-init` |
| 3 | Plan (PRD + architecture) | `*agent pm`, `*agent architect` |
| 4 | Write specs | `/opsx:propose <feature>`, `/opsx:ff` |
| 5 | Impact analysis | GitNexus MCP queries + `/graphify path A B` |
| 6 | Brainstorm + plan tasks | `/superpowers:brainstorm`, `/superpowers:write-plan` |
| 7 | Execute (TDD) | `/superpowers:execute-plan` вҖ” 1 commit per task |
| 8 | 3-layer self-review | See below |
| 9 | Push PR | `git push` вҖ” pre-push hook auto-runs verify + scan |
| 10 | Archive + ship | `make archive`, `make mine`, tag, deploy |

See the [full walkthrough](docs/onboarding/full-guide.md#5-your-first-project--a-walkthrough) for a concrete example.

## 3-Layer Review (Mandatory Before PR)

| Layer | What It Checks | Command |
|---|---|---|
| 1. Methodology | Code vs. plan, standards, architecture | `/superpowers:code-review` |
| 2. Spec compliance | Code vs. OpenSpec specifications | `make verify` |
| 3. Knowledge graph sync | Re-index so the graph reflects your changes | `make index` |

If Layer 2 fails, go back and fix the code before pushing.
